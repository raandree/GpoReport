# Project Brief: GPO Report Search System

## Core Mission
Create PowerShell scripts to search Group Policy Object (GPO) settings using wildcard patterns, returning structured information about setting locations, GPO context, and policy hierarchy.

## Problem Statement
System administrators and security professionals need to efficiently search through Group Policy settings across multiple GPO XML files to:
- Conduct security audits and compliance checks
- Troubleshoot policy configuration issues
- Discover configuration patterns across environments
- Verify specific settings are properly configured

## Primary Requirements

### R1: Dual XML Format Support
- **PowerShell XMLs**: Files generated by `Get-GPO | Get-GPOReport`
- **GPMC Report XMLs**: Files generated by GPMC "Generate Report" feature
- Handle different XML structures and namespaces

### R2: Comprehensive Search Capabilities
- Wildcard pattern matching (`*password*`, `Enable*`, etc.)
- Case-sensitive and case-insensitive search modes
- Search both text content and XML attributes
- Multi-file and directory processing with recursion

### R3: Accurate Category Detection
- **Security Settings**: Account Policies, Local Policies, Advanced Audit Configuration
- **Administrative Templates**: Full hierarchy path extraction
- **Group Policy Preferences**: Environment Variables, Registry, Files, etc.
- **Security Options Subcategorization**: Domain Controller, Devices, Other

### R4: Structured Output Format
For each match, return:
- GPO Information (DisplayName, DomainName, GUID, timestamps)
- Category Path (hierarchical location in policy structure)
- Setting Details (name, value, context)
- Source file information

### R5: Mapping Table Validation
Must correctly categorize 25+ specific search patterns according to provided mapping table:
- "Chile" → "Security Settings > Local Policies > User Rights Assignment"
- "LDAP server signing requirements" → "Security Settings > Local Policies > Security Options > Domain Controller"
- "Turn off notifications network usage" → "Administrative Templates > Start Menu and Taskbar > Notifications"
- And 22+ additional mappings

## Success Criteria
1. **100% Mapping Table Validation**: All 25+ patterns return correct category paths
2. **Automated Test Coverage**: Comprehensive Pester test suite with 100% pass rate
3. **Real-World XML Compatibility**: Handle diverse XML files with encoding variations
4. **Performance**: Process large XML files efficiently
5. **Usability**: Clear parameter validation, helpful error messages, verbose output

## Technical Constraints
- **PowerShell Environment**: Must run in Windows PowerShell and PowerShell Core
- **No External Dependencies**: Use only built-in .NET XML processing
- **Encoding Resilience**: Handle UTF-16 declarations with UTF-8 content
- **Namespace Awareness**: Properly handle XML namespaces in GPMC reports

## Out of Scope
- Real-time GPO monitoring
- GPO modification or creation
- Integration with Active Directory APIs
- Graphical user interface
- Reporting beyond console output
