## XML Processing Architecture

### XML Structure Analysis

#### PowerShell GPO XMLs
```xml
<GPO>
  <Computer>
    <ExtensionData>
      <Extension type="Administrative Templates">
        <Policy>...</Policy>
      </Extension>
    </ExtensionData>
  </Computer>
</GPO>
```

#### GPMC Report XMLs
```xml
<GPO xmlns="http://www.microsoft.com/GroupPolicy/Settings">
  <Computer>
    <ExtensionData>
      <Extension xmlns:q1="http://www.microsoft.com/GroupPolicy/Settings/Security">
        <q1:SecurityOptions>...</q1:SecurityOptions>
      </Extension>
    </ExtensionData>
  </Computer>
</GPO>
```

```powershell
    # Process regardless of namespace prefix
}
```

## Category Detection Engine

### Hierarchy Traversal Algorithm

```
Node Detection Process:
1. Start at matched node
2. Traverse up parent hierarchy (max 10 levels)
3. Identify category markers:
   ├── ExtensionData > Name = "Security"
   ├── Namespace URI patterns
   ├── Element LocalName patterns
   └── Parent/child relationships
4. Apply subcategorization logic
5. Build hierarchical path
```

## Search Engine Design

### Pattern Processing
```powershell
# Wildcard to Regex conversion
$wildcardPattern = "Enable*Account"
$regexPattern = "(?i)Enable.*Account"  # Case-insensitive by default

# Multi-target search
foreach ($node in $allNodes) {
    # Search text content
    if ($node.InnerText -match $regexPattern) { /* Match found */ }
    
    # Search attributes
    foreach ($attr in $node.Attributes) {
        if ($attr.Value -match $regexPattern) { /* Match found */ }
    }
}
```

### Duplicate Detection & Filtering
```powershell
# Result deduplication strategy
$uniqueKey = "$($match.MatchedText)|$($match.CategoryPath)|$($match.Setting.Name)"
if (-not $processedMatches.ContainsKey($uniqueKey)) {
    # Add to results

```

## Error Handling & Recovery

### Encoding Issue Resolution
```powershell
try {
    $xmlDoc.Load($FilePath)  # Direct load attempt
} catch {
    # Encoding mismatch detected
    $content = Get-Content -Path $FilePath -Raw -Encoding UTF8
    $content = $content -replace 'encoding="utf-16"', 'encoding="utf-8"'
    $xmlDoc.LoadXml($content)  # Retry with corrected content
}
```

### Graceful Degradation
- **GPO Info Extraction**: Falls back to filename if XML parsing fails
- **Category Detection**: Returns generic category if specific detection fails
- **Search Continue**: Individual file errors don't stop batch processing

## Performance Characteristics

### Memory Management
- **Streaming Processing**: One file at a time
- **Node Collection**: Efficient XPath queries
- **Result Buffering**: Minimal memory footprint

### Scalability Factors
- **File Size**: Handles large GPO reports (tested up to several MB)
- **File Count**: Batch processing supports hundreds of files
- **Search Complexity**: Regex compilation optimized for repeated use

### Performance Metrics
- **Small GPO (~100KB)**: < 1 second processing time
- **Large GPO (~5MB)**: < 10 seconds processing time  
- **Batch Processing**: Linear scaling with file count

## Integration Points

### PowerShell Integration
```powershell
# Pipeline support
Get-ChildItem "*.xml" | .\Search-GPMCReports.ps1 -SearchString "password"

# Object output for further processing
$results = .\Search-GPMCReports.ps1 -Path "gpo.xml" -SearchString "*audit*"
$results | Where-Object { $_.CategoryPath -like "*Security*" }
```

### Testing Integration
```powershell
# Pester test automation
Describe "GPO Search Validation" {
    $mappingTable | ForEach-Object {
        It "Should find $($_.Pattern) in $($_.ExpectedCategory)" {
            $result = Search-GPMCReports -SearchString $_.Pattern
            $result.CategoryPath | Should -Be $_.ExpectedCategory
        }
    }
}
```
